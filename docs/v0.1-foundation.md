# v0.1 — Foundation & LCD theme

**Goal:** Strip the create-next-app boilerplate and stand up the project skeleton,
the LCD palette, retro fonts, and dependencies — so every later phase has a clean,
authentic base to build on. No games yet.

## Prerequisite reading

- `node_modules/next/dist/docs/01-app/` — App Router, layouts, metadata, fonts in
  Next.js 16. Confirm `next/font` API and any deprecations before editing
  `app/layout.tsx`.

## Scope

- Remove boilerplate from `app/page.tsx`, `app/layout.tsx`, `app/globals.css`
  (Next logo, Geist fonts, marketing copy, demo styles).
- Add the LCD palette as CSS variables and remove the light/dark `@media` theme —
  the device is always the same greenish-gray.
- Add retro fonts: **Press Start 2P** (labels/digits) and **VT323** (panel text),
  via `next/font/google`. Confirm the import style against the installed docs.
- Install **Zustand** (`npm i zustand`) — required by the engine in v0.2.
- Create empty directories with `.gitkeep` to lock in the target structure:
  `components/`, `lib/engine/`, `games/{tank,shooting,fillShot}/`,
  `assets/{sounds,fonts}/`.
- Home page (`app/page.tsx`): a simple LCD-styled menu listing the three games
  (links to `/shooting`, `/tank`, `/fill-shot` — routes can be placeholders).

## Deliverables

- `app/globals.css` exposing the 4 LCD variables + base `16px` cell token.
- `app/layout.tsx` loading retro fonts and setting `<title>` to the device name.
- A minimal LCD-styled landing menu.
- `zustand` in `package.json` dependencies.
- Directory skeleton committed.

## Out of scope

- Canvas rendering, game loop, any gameplay, the handheld shell chrome.

## Acceptance criteria

- [ ] No remaining create-next-app assets or copy.
- [ ] Page renders only in the 4 LCD colors; no gradients/shadows/dark-mode.
- [ ] Retro fonts load and render.
- [ ] `zustand` installed; `npm run build` passes.
- [ ] Directory structure matches ROADMAP target.

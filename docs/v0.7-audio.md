# v0.7 — Audio

**Goal:** Replace the sound stubs left in v0.4–v0.6 with real piezo-style beeps that
sound like a cheap handheld speaker. One channel, no music.

## Design

- Emulate a single-channel **piezo buzzer** — short square-wave beeps via the Web
  Audio API (oscillator + gain), not sampled audio files where avoidable.
- **Single channel:** a new beep cuts off the previous one (authentic — the real
  device can't mix).
- **No background music** during gameplay.
- Low fidelity on purpose: square waves, abrupt envelopes, narrow pitch range.

## Effects to implement

| Event      | Feel                          |
| ---------- | ----------------------------- |
| Shoot      | very short high blip          |
| Hit        | short low buzz                |
| Level up   | quick rising 2–3 note arp     |
| Pause      | single neutral beep           |
| Game over  | descending low tone           |

## Integration

- A small `sound` module (e.g. `lib/engine/sound.ts` or `assets/sounds/`) exposing
  named effects; games call these at the existing stub sites.
- Respect the global mute toggle (`M` key / sound button) from the store; persist
  the mute preference in `localStorage`.
- Initialize/resume the AudioContext on first user gesture (browser autoplay rules
  — verify against current behavior).

## Out of scope

- Music, stereo, reverb, multi-voice mixing, volume sliders.

## Acceptance criteria

- [ ] Each listed event produces its beep across all three games.
- [ ] Only one sound plays at a time (single channel).
- [ ] Mute toggle silences everything and persists across reloads.
- [ ] No audio without a prior user gesture; no console autoplay errors.
- [ ] No background music.

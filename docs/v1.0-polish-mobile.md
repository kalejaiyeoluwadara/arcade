# v1.0 — Polish, mobile & persistence

**Goal:** Ship-ready. Make the handheld fully playable on touch devices, lock in
persistence and difficulty, add the final LCD-authenticity touches, and do a
cross-game consistency pass.

## Scope

### Mobile controls

- Wire the v0.3 on-screen **D-pad** and **A/B / Start / mute** buttons to the engine
  input layer for touch (parity with keyboard).
- Large, tactile hit targets; instant press states (no modern animations).
- **Haptic feedback** via the Vibration API where available (short buzz on
  fire/hit), gracefully ignored where not.
- Layout: device fills the viewport comfortably on phones; no horizontal scroll.

### Persistence & difficulty

- Per-game **high scores** in `localStorage`, surfaced as `HI-SCORE`.
- **Difficulty levels** selectable and remembered; verify each game consumes them
  (tick rate + spawn/target).
- Remember last mute state and last played mode.

### Authenticity polish

- Optional **screen-refresh flicker** every few frames (subtle, toggleable).
- Faint "off-segment" LCD ghosting around active cells.
- Final audit: confirm no easing/interpolation/particles/extra colors crept in
  across all three games.

### Consistency pass

- Uniform game-over / pause / start overlays across games.
- Shared HUD wording and behavior; mode label correct per game.

## Out of scope

- Online features, accounts, leaderboards, new games, cosmetics.

## Acceptance criteria

- [ ] All three games fully playable by touch on a phone.
- [ ] Haptics fire where supported; no errors where not.
- [ ] High scores, difficulty, and mute persist across reloads, per game.
- [ ] Optional flicker/ghosting works and can be turned off.
- [ ] Authenticity audit passes for every game (4 colors, grid-only, no smooth motion).
- [ ] `npm run build` clean; manually verified on desktop + mobile viewport.

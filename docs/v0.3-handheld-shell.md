# v0.3 — Handheld shell UI

**Goal:** The physical handheld around the screen. After this phase the device
*looks* real and complete (plastic body, LCD window, D-pad, A/B buttons, score
panel) even though the screen still shows only the v0.2 demo.

## Scope — components

- **`HandheldShell.tsx`** — rounded plastic body that frames everything: LCD
  centered, D-pad left, A/B right, Start/Pause + sound toggle. Pure CSS/markup;
  responsive so it scales on small screens (full touch wiring is v1.0).
- **`LCDScreen.tsx`** — the bezel + greenish LCD window that hosts the canvas. Adds
  the authentic "always-on" faded segment look around the active area.
- **`DPad.tsx`** — four-direction pad. Emits the same input events the engine's
  keyboard handler uses, so games don't care about the source.
- **`ActionButtons.tsx`** — A (fire) and B, plus Start/Pause and mute. Visual press
  states, no easing animations.
- **`ScorePanel.tsx`** — the info panel using a segmented digital font.

## Information panel must show

`HI-SCORE` · `SCORE` · `LEVEL` · `LIVES` · `GAME MODE` · `PAUSE` indicator —
reading from the Zustand store so values are live.

## Style constraints

- Plastic shell may use real-world plastic shading (it's the physical device, not
  the screen). **Inside the LCD window**, the authenticity rules are absolute: 4
  colors only, no shadows/gradients/transparency.
- Segmented/retro fonts only (Press Start 2P / VT323), no modern typography.
- Button presses are instant state swaps — no smooth modern animations.

## Deliverables

- All five components under `/components`, composed into a single device view.
- D-pad and buttons dispatch into the engine input layer (parity with keyboard).
- Score panel bound to the store.

## Out of scope

- Game-specific HUD, audio, mobile haptics, real gameplay.

## Acceptance criteria

- [ ] Device renders as a believable handheld with all controls present.
- [ ] LCD window strictly obeys the 4-color rule.
- [ ] On-screen D-pad/buttons drive the v0.2 demo identically to the keyboard.
- [ ] Score panel reflects live store values.
- [ ] Layout holds up from phone width to desktop.
